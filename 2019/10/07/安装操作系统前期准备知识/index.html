安装操作系统前期准备知识
Created 2019-10-07| Updated 2020-02-01|Linux
  1. Linux系统就是安装在磁盘。
  2. 各个组件或设备在Linux下面都是一个文件，几乎所有的硬件设备文件都在/dev这个目录内
  3. 选择与Linux搭配的主机配置：RAM内存越大越好。事实上在Linux服务器中，内存的重要性比CPU还要高得多。因为如果内存不够大，就会使用到硬盘的内存交换空间(swap)。硬盘比内存的速度要慢得多，所以内存小可能会影响到整体系统的性能。
  4. 常见的设备与其在Linux当中的文件名：IDE硬盘(/dev/hd[a-d])；SCSI/SATA/USB硬盘(/dev/sd[a-p])；U盘(/dev/sd[a-p]{与SATA相同})；软驱(/dev/fd[0-1])。更多
  5. 与IDE接口不同，SATA/USB接口的磁盘根本没有一定的顺序，根据Linux内核检测到磁盘的顺序决定它的设备文件名。
  6. 磁盘的第一个扇区主要记录两个重要信息：
    • 主引导分区(Master Boot Record, MBR):可以安装引导加载程序的地方，446bytes。{MBR是很重要的，因为当系统在开机的时候会主动去读取这个区块的内容，这样系统才会知道你的程序放在哪里且该如何进行开机}
    • 分区表(partition table):记录整块硬盘分区的状态，有64bytes。
  7. 其实所谓的“分区”只是针对那个64bytes的分区表进行设置而已。
  8. 硬盘默认的分区表仅能写入四组分区信息。
  9. 这四组分区信息我们称为主(Primary)或扩展(Extended)分区。
  10. 分区的最小单位为柱面(cylinder)。
  11. 分区表中1-4号都是保留给Primary或Extended的，所以逻辑分区的设备名称号码就由5号开始了。
  12. 主分区与扩展分区最多可以有四个(硬盘的限制)。
  13. 扩展分区最多只能有一个(操作系统的限制)。
  14. 逻辑分区是由扩展分区持续切割出来的分区。
  15. 能够被格式化后作为数据访问的分区为主分区与逻辑分区，扩展分区无法格式化。
  16. 如果扩展分区被破坏，所有逻辑分区将会被删除。
  17. 开机流程：BIOS->MBR->boot loader->内核文件
  18. boot loader的功能：提供菜单、加载内核、转交控制权给其他loader
  19. boot loader可以安装在MBR或boot sector(每个分区的引导分区)
  20. loader只会认识自己的系统分区内的可开机内核文件，以及其他loader而已。
  21. Linux操作系统的文件使用目录树系统，与磁盘的对应需要有“挂载”的操作才行。
  22. 所谓的“挂载”就是利用一个目录当成进入点，将磁盘分区的数据放置在该目录下；也就是说，进入该目录就可以读取该分区的意思。
  23. 分区时，将启动扇区所在的分区规范在小于1024个扇面即可。技巧：将/boot放在开始。
  24. swap内存转换空间的功能：当有数据被存放在物理内存里面，但是这些数据又不是常被CPU所取用时，那么这些不常被使用的程序将会被丢到硬盘的swap交换空间当中，而将速度较快的物理内存空间释放出来给真正需要的程序使用。 一般要求swap应该是1.5~2倍的物理内存量。
