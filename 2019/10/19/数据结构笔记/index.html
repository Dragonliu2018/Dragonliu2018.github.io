数据结构笔记
Created 2019-10-19| Updated 2020-03-04|Course

第1章 绪论

Key

  • 基本概念：数据、数据元素、数据对象、数据结构、结构(集合、线性结构、树形结构、图/网状结构)、逻辑结构、物理/存储结构、顺序映像、非顺序映像、顺序存储结构、非顺序存储结构、数据类型、抽象数据类型(ADT)
  • 算法
    • 重要特性：有穷性、确定性、可行性、输入、输出
    • 设计要求：正确性、可读性、健壮性、效率与低存储量需求
    • 效率度量：时间复杂度、空间复杂度

Value

  1. 在相同的数据规模下，复杂度为O(n)的算法在时间上总是优于复杂度为O(2n)的算法。
  2. 算法的时间复杂度取决于问题的规模以及待处理数据的初态。

第2章 线性表

Key

  • List: SqList、LinkList、SLinkList、CirLinkList、DuLinkList
  • 题目: 线性表的就地逆置(SqList：O(L)、LinkList: O(L))、报数出列(SqList、CirLinkList)、学生排队(DuLinkList)、线性表的插入排序(SqList: O(n2)、LinkList: O(n2))、递增线性表有序交集(SqList、LinkList)、折点数(DuLinkList)、中间数(SqList)、删除线性表i个之后的k个元素(SqList)、非递减线性表删除相同元素(LinkList)、合并&交&前驱&移动(CirLinkList)、删除零元素(SqList)

Value

  1. 链式存储的优点是可以随机存储。(错误)
  2. 顺序表适宜于随机存储，而链表适宜于顺序存储。
  3. 静态链表中的”指针”表示的是下一元素在数组中的下标。

第3章 栈和队列

Key

  • Stack、SqStack、数制转化、括号匹配的检验、行编辑程序、迷宫求解、表达式求值、Hanoi塔
  • Queue：LinkQueue、SqQueue(circle)
  • 题目：前/中/后缀表达式转化(加括号法、栈法)、进/出队&判断对空(循环链表表示队列)、两个栈实现队列、括号配对、八皇后问题、窗口优先级、单向循环链表改双向、双向循环链表取奇项(O(n))、按频度排序(双向循环链表)

Value

  1. 循环队列(顺序存储)：
    • 入队：Q.rear = (Q.rear+1)%MAXQSIZE
    • 出队：Q.front = (Q.front+1)%MAXQSIZE
    • 判断已满：(Q.rear+1)%MAXQSIZE == Q.front
    • 元素个数：Q.length = (Q.rear-Q.front+MAXQSIZE)%MAXQSIZE

第4章 串

Key

  • 术语：串、长度、空串、子串、主串、位置、相等、空格串、
  • String：SString、HString、LString

Value

  1. 某串的长度小于一个常数，则采用顺序存储方式最节省空间。
  2. 串是一种数据对象特殊(char)的线性表。

第5章 数组和广义表

Key

  • Array
  • 矩阵的压缩存储
    • 特殊矩阵
    • 稀疏矩阵：三元组顺序表、行逻辑链接的顺序表、十字链表
  • GList

Value

  1. 对矩阵进行压缩存储后，稀疏矩阵会失去随机存储的优点。
  2. 任何一个非空列表其表头可能是原子，也可能是列表，而表尾必为列表。
    • 列表(): 空表，长度n为0；
    • 列表(()): 长度n为1，可分解其表头表尾均为空表()。

第6章 树和二叉树

Key

  • 术语：子树、度、叶子/终端结点、分支结点/非终端结点、层次、深度、有/无序树、森林、二叉树性质(5)、满二叉树、完全二叉树、遍历二叉树、线索二叉树、哈夫曼树/最优二叉树、哈夫曼编码
  • Tree：BinaryTree

Value

  1. 树的结点数 = 所有结点度数之和+1 = 各度数结点数之和
  2. 哈夫曼树是正则二叉树(无度数为1的结点)，则n0=n2+1 n=n0+n2 -> n=2n0-1 -> n0=(n+1)/2;叶结点n0对应的即是不同的编码。
  3. 若一个具有n个结点，k条边的无向图是一个森林(n>k)，则该森林必有n-k棵树。
  4. 一个具有N个结点的二叉树采用二叉树采用二叉链表进行存储，其中空指针域为N+1个；->(每个结点有2个链域 -> 2N个，除根结点外，每个结点只能被指1次 -> N-1不为空)。

第7章 图

Key

  • 术语：
    • 无向图：顶点、边、完全图、稀疏图、稠密图、权、网、子图、邻接点、度、路径、回路/环、简单路径、简单回路/环、连通图、连通分量、生成树
    • 有向图：顶点、弧、弧头、弧尾、有向完全图、稀疏图、稠密图、权、网、子图、出度、入度、路径、回路/环、简单路径、简单回路/环、强连通图、强连通分量、弱连通图
  • Graph：MGraph、ALGraph、OLGraph、AMLGraph
  • 题目：

Value

  1. 设图G有n个顶点和e条边，采用邻接表存储，则拓扑排序的时间复杂度O(n+e)。
  2. 图的邻接矩阵存储所占用空间大小只与顶点个数有关，更准确 -> 设顶点数n, 则与n2成正比。
  3. 具有n个顶点的有向图，要连通全部顶点，至少需要n条弧，无向图n-1条弧。
  4. 关键路径是事件结点网络中从源点到汇点的最长路径。
  5. 图可以没有边，但不能没有顶点。(错误)

Key

  • StaticSearch(静态查找表)
    • 顺序表的查找—顺序查找
    • 有序表的查找—折半查找
    • 静态树表的查找
    • 索引顺序表的查找
