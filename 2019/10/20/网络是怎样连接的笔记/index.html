网络是怎样连接的笔记
Created 2019-10-20| Updated 2019-10-21|Reading Notes

第一章 浏览器生成消息(Web浏览器)

章节

  • 生成HTTP请求信息
  • 向DNS服务器查询Web服务器的IP地址
  • 全世界DNS服务器的大接力
  • 委托协议栈发送信息

    术语

  1. URL：Uniform Resource Locator 统一资源定位符
  2. FTP: File Transfer Protocol 文本传送协议
  3. HTTP: Hypertext Transfer Protocol 超文本传送协议
  4. DNS：Domain Name System 域名服务系统
  5. 协议栈：操作系统内部的网络控制软件，也叫“协议驱动” “TCP/IP驱动”等。
  6. UDP：User Datagram Protocol 用户数据报协议

    KEY

  7. 1条请求信息中只能写1个URL。如果需要获取多个文件，必须对每个文件单独发送1条请求。
  8. IP地址的主机号：全0表示整个子网，全1表示向子网上所有设备发送包，即“广播”。
  9. Socke库是用于调用网络功能的程序组件集合。
  10. 根据域名查IP地址时，浏览器会使用Socket库中的解析器。
  11. DNS服务器会从域名与IP地址的对照表中查找相应的记录，并返回IP地址。
  12. 在域名中，越靠右的位置表示其层次越高；下级的域称为子域。
  13. 向操作系统内部的协议栈发出委托时，需要按照指定的顺序来调用Socket库中的程序组件。
  14. 收发数据的操作：
    • 创建：创建套接字
    • 连接：将管道连接到服务器端的套接字上
    • 通信：收发数据
    • 断开：断开管道并删除套接字
  15. 应用程序是通过“描述符”这一类似号码牌的东西来识别套接字的。
  16. IP地址不是分配给每个设备，而是分配给设备中安装的网络硬件的。
  17. 描述符：应用程序用来识别套接字的机制
  18. IP地址和端口号：客户端和服务器之间用来识别对方套接字的机制
  19. 向DNS服务器发送请求消息的程序是解析器。
  20. URL中的www是Web服务器上的一种命名；World Wide Web，中文名字为“万维网”，”环球网”等，常简称为Web。

第2章 用电信号传输TCP/IP数据(协议栈、网卡)

章节

  • 创建套接字
  • 连接服务器
  • 收发数据
  • 从服务器断开并删除套接字
  • IP与以太网的包收发操作
  • UDP协议的收发操作

    术语

  1. MAC：Media Access Control / MAC地址：符合IEEE规格的局域网设备都使用统一格式的地址。
  2. PID：Process ID 进程标识符
  3. MTU：Maximum Transmission Unit 最大传输单元
  4. MSS：Maximum Segment Size 最大分段大小
  5. SFD：Start Frame Delimiter 起始帧分界符
  6. FCS：Frame Check Sequence 帧校验序列
  7. SUN：Synchronize 同步
  8. ACK：Acknowledge character 确认字符（在数据通信中，接收站发给发送站的一种传输类控制字符。表示发来的数据已确认接收无误）
  9. ARP：Address Resolution Protocol 地址解析协议
  10. CRC：Cyclic Redundancy Check 循环冗余校验
  11. MAU：Medium Attachment Unit 介质连接单元
  12. PHY：Physical Layer Device 物理层设置
  13. PnP：Plug and Play 是一种自动对扩展卡和周边设备进行配置的功能。

    KEY

  14. 浏览器、邮件等一般应用程序收发数据用TCP；DNS查询等收发较短的控制数据时用UDP。
  15. IP中还包含ICMP协议和ARP协议.ICMP用来告知网络包传送过程中产生的错误以及各种控制信息，ARP用来根据IP地址查询相应的以太网MAC地址。
  16. 协议栈是根据套接字中记录的控制信息来工作的。
  17. 创建套接字时，首先分配一个套接字所需的内存空间，然后向其中写入初始状态。
  18. 通信操作中使用的控制信息分为两类：头部中记录的信息、套接字(协议栈中的内存空间)中记录的信息。
  19. 连接操作的第一步是在TCP模块处创建表示连接控制信息的头部。
  20. 通过TCP头部中的发送方和接收方端口号可以找到要连接的套接字。
  21. MTU：一个网络包的最大长度，以太网中一般为1500字节；MSS：除去头部后，一个网络包所能容纳的TCP数据的最大长度。
  22. 通过“序号”和“ACK号”可以确认接收方是否接收到了网络包。
  23. 能够接收的最大数据量称为窗口大小，一般和接收方的缓冲区大小一致。
  24. 集线器是按照以太网规则传输包的设备，路由器是按照IP规则传输包的设备。
  25. IP模块负责添加两个头部：MAC头部(以太网用的头部，包含MAc地址)、IP头部(IP用的头部，包含IP地址)。
  26. 无论要收发的包是控制包还是数据包，IP对各种类型的包的收发操作都是相同的。
  27. IP头部的“接受方IP地址”填写通信对象的IP。发送方IP地址需要判断发送所使用的网卡，并填写该网卡的IP地址。
  28. 子网掩码：用来判断IP地址中网络号与主机号分界线的地址。
  29. IP模块根据路由表Gateway栏的内容判断应该把包发送给谁。
  30. 以太网3个性质：将包发送到MAC头部的接收方MAC地址代表的目的地、用发送方MAC地址识别发送方、用以太类型识别包的内容。
  31. 网卡的ROM中保存着全世界唯一的MAC地址，这是在生产网卡时写入的。
  32. 网卡中保存的MAC地址会由网卡驱动程序读取并分配给MAC模块。
  33. 每个包的前面都有报头和SFD，报头用来测定时机，SFD用来确定帧的起始位置。
  34. 网卡的MAC模块生成通用信号，然后由PHY(MAU)模块转换成可在网线中传输的格式，并通过网线发送出去。
  35. 端口号用来指定服务器程序的种类，位于TCP头部。
  36. TCP对包是否正确送达进行确认。
  37. 在收到ACK号之前继续发送下一个包的方式是滑动窗口方式。
